# Power BI Dashboard Plan

## Dashboard Objective

Answer the business question:

Which infrastructure storage clusters are likely to hit utilization risk thresholds in the next 3 to 6 months, and what action should leadership take?

The dashboard should present workload-derived capacity pressure, simulated enterprise storage inventory, forecasted utilization, model confidence context, and action-oriented risk recommendations.

Because the available Kaggle extraction contains one month folder, forecasting is performed on daily cluster-level capacity history and then summarized into 30-day, 90-day, and 180-day planning windows.

## Suggested Measures

Create these measures in Power BI:

```DAX
Total Clusters = DISTINCTCOUNT(pbi_capacity_risk_summary[cluster_name])

Critical Clusters =
CALCULATE(
    DISTINCTCOUNT(pbi_capacity_risk_summary[cluster_name]),
    pbi_capacity_risk_summary[risk_level] = "Critical"
)

High Risk Clusters =
CALCULATE(
    DISTINCTCOUNT(pbi_capacity_risk_summary[cluster_name]),
    pbi_capacity_risk_summary[risk_level] = "High"
)

Total Usable TB = SUM(pbi_capacity_risk_summary[usable_capacity_tb])

Total Current Used TB = SUM(pbi_capacity_risk_summary[current_used_tb])

Portfolio Utilization % =
DIVIDE([Total Current Used TB], [Total Usable TB]) * 100
```

## Formatting Guidance

- Use red for Critical, orange for High, yellow for Medium, and green or neutral gray for Low.
- Keep utilization percentages in `0-100` format.
- Sort risk tables by `risk_level`, then `forecast_180d_utilization_pct` descending.
- Use slicers consistently across pages.
- Keep a visible label or note that inventory fields are simulated.

## Dashboard Acceptance Criteria

- The first page answers which clusters need action.
- Users can filter by datacenter, business unit, platform, and risk level.
- Forecast visuals show 80%, 85%, and 90% thresholds.
- Forecast visuals use daily forecast dates and can be summarized into 30-day, 90-day, and 180-day windows.
- Risk recommendations are visible without drilling.
- Simulated inventory is explicitly labeled.
- No visual implies that simulated fields came from the raw Bitbrains dataset.
